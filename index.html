1. Explain in detail stages of Edge detection process with block diagram. (8)

Answer

Edge detection is done in three stages:

  1. Filtering
  1. Differentiation
  1. Localization

Filtering

  • It is better to filter the input image to get maximum performance for the edge detectors. This stage may be performed either explicitly or implicitly.
  • It involves smoothing, where the noise is suppressed without affecting the true edges.
  • In addition, this phase uses a filter to enhance the quality of the edges in the image. Normally, Gaussian filters are used as they are proven to be very effective for real-time images.

Differentiation

  • This phase distinguishes the edge pixels from other pixels.
  • The idea of edge detection is to find the difference between two neighbourhood pixels.
  • If the pixels have the same value, the difference is zero. This means that there is no transition between the pixels. The non zero difference indicates the presence of an edge point.

2. Explain the steps in automatic image analysis and interpretation. (6)

Answer

Automatic

  • Automatic segmentation algorithms are a preferred choice as they segment the structure of objects without any human intervention.
  • Automatic segmentation are preferred if the tasks need to be carried out for a large number of images.

Semi - Automatic

  • Semi-automatic algorithms are a combination of automatic and manual algorithms.
  • In semi - automatic algorithms, human intervention is required in the initial stages.
  • Normally, the human observer is supposed to provide the initial seed points indicating the ROI. Then the extraction process is carried out automatically as dictated by the logic of the segmentation algorithm.

Based on Pixel Relationships

1. Contextual (region-based or global) Algorithms

  • Contextual algorithms group pixels together based on common properties by exploiting the relationships that exist among the pixels.
  • These are also known as region-based or global algorithms.
  • In region-based algorithms, the pixels are grouped based on some sort of similarity that exists between them.

2. Non - Contextual (pixel-based or local) Algorithms

  • Non-contextual algorithms are also known as pixel-based or local algorithms.
  • These algorithms ignore the relationship that exists between the pixels or features.
  • The idea is to identify the discontinuities that are present in the image such as isolated lines and edges. These are then simply grouped into a region based on some global-level property.
  • Intensity-based thresholding is a good example of this method.

3. Explain Canny edge detection algorithm. (6)

Answer

Canny Edge Detection is a popular edge detection algorithm used in computer vision and image processing to identify the edges or boundaries within an image.

Developed by John F. Canny in 1986, this algorithm is known for its robustness and accuracy in detecting edges.

Steps of the Canny Edge Detection Algorithm

1. Noise Reduction

  • Since edge detection is susceptible to noise, the first step is to reduce noise in the image using a Gaussian filter.
  • The Gaussian filter smooths the image by averaging out variations in intensity, which helps in reducing the effect of noise.

3. Non-Maximum Suppression

  • The purpose of this step is to thin out the edges, so only the pixels that constitute the edge remain, and all other pixels are suppressed.
  • For each pixel, the algorithm checks the pixel's gradient direction and compares its intensity with the neighboring pixels in the same direction. If the pixel's intensity is not the maximum, it is suppressed (set to zero).
  • This step ensures that the edges are one pixel thick, making the edge detection more precise.

4. Double Threshold

  • After non-maximum suppression, the edge pixels are categorized based on their intensity using two threshold values: a high threshold and a low threshold.
  • Strong edges: Pixels with gradient magnitude greater than the high threshold are considered as strong edges.
  • Weak edges: Pixels with gradient magnitude between the low and high thresholds are considered as weak edges.
  • Non-edges: Pixels with gradient magnitude below the low threshold are suppressed (set to zero).

5. Edge Tracking by Hysterisis

  • This step finalizes the edge detection process. It involves connecting the weak edges to the strong edges if they are adjacent to each other. Weak edges that are connected to strong edges are considered part of the edge, while the rest are discarded.
  • This step ensures that only significant edges are retained, reducing the likelihood of false edges.

6. Explain Edge detection with an example. List the types of edges. (6)

Answer

  • An edge is a set of connected pixels that lies on the boundary between two regions that differ in grey value.
  • The pixels on an edge are called edge points.
  • Edge detection is important as it outlines objects and represents significant intensity transitions.

An edge is typically extracted by computing the derivative of the image function. This involves:

  1. Magnitude of the derivative: Indicates the strength or contrast of the edge.
  1. Direction of the derivative vector: Measures edge orientation.

Types of Edges

  • Step Edge: An abrupt intensity change.
  • Ramp Edge: A gradual change in intensity.
  • Spike Edge: A quick change that immediately returns to the original intensity level.
  • Roof Edge: Not instantaneous over a short distance.

9. Explain about different types of edge detectors. (6)

Answer

The different types of edge detectors in image processing are:

  1. Gradient Filters (Derivative Filters):
    • Use the differentiation technique to detect edges.
    • They detect edges in all directions (isotropic).
    • Common examples include Sobel, Prewitt, and Roberts operators.
  1. Template Matching Filters:
    • Use templates that resemble the target shapes and match them with the image.
    • The template is rotated in eight directions, producing compass masks.
    • Sensitive to direction and used for tasks like point and line detection.
  1. Gaussian Derivatives:
    • Very effective for real-time images.
    • Used along with derivative filters.
    • Help in smoothing and reducing noise before edge detection.
  1. Pattern Fit Approach:
    • Considers a surface as a topographic surface with pixel values representing altitude.
    • Aims to fit a pattern over a neighborhood of a pixel to calculate edge strength.
    • Properties of edge points are calculated based on the parameters.
